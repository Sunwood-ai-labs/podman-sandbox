# 💻 CTO会議ルール (アジャイル方式)

## アジャイル原則

1. **スプリント**: 各会議を1スプリントとして扱う
2. **バックログ**: アクションアイテムをバックログとして管理
3. **インクリメント**: 毎回小さな成果物を出力
4. **振り返り**: 改善点を常に意識

## スプリント構成

### 1. スプリントプランニング (5分)
- 今回のスプリントゴールを設定
- 着手するタスクを選択

### 2. 実行 (15分)
- 実際のタスク遂行
- コードレビュー、調査、ドキュメント作成など

### 3. スプリントレビュー (5分)
- 成果物の確認
- デモンストレーション

### 4. レトロスペクティブ (5分)
- 何がうまくいったか
- 何を改善できるか

## 出力形式

```markdown
## 🎯 スプリントゴール
今回のスプリントで達成する目標

## 📋 スプリントバックログ
- [ ] タスク1 (見積: S/M/L)
- [ ] タスク2 (見積: S/M/L)

## ✅ 完了したタスク
- [x] 完了タスク

## 📦 インクリメント (成果物)
- 具体的な成果

## 🔍 レトロスペクティブ
### Keep (続けること)
### Problem (問題点)
### Try (次回試すこと)

## 🚫 ブロッカー
- 阻害要因があれば報告
```

## アジャイル价值观

- **顧客との協調**: ユーザー視点を常に意識
- **変化への対応**: 柔軟に計画を変更
- **動くソフトウェア**: 実際に動く成果物を優先
- **継続的な改善**: 毎回少しずつ良くする
