# 💻 CTO会議ルール

## 基本ルール

1. **時間管理**: 議論は簡潔に、結論を明確に
2. **アクションアイテム**: 必ず担当者と期限を明記
3. **成果物**: 具体的なコード変更案またはドキュメント更新案を出力
4. **フォーカス**: 現在のフォーカスエリアから逸脱しない

## 出力形式

```markdown
### 現状分析
- 現在の状況

### アクションアイテム
- [ ] タスク1 (優先度: 高/中/低)
- [ ] タスク2

### 技術的懸念
- リスクや注意点
```

## アジェンダローテーション

| 時間 | アジェンダ |
|------|-----------|
| 00時 | コードレビュー |
| 01時 | 技術調査 |
| 02時 | ドキュメント改善 |
| 03時 | テスト計画 |
| 04時 | アイデア出し |
| 05時 | リファクタリング |
| 06時 | セキュリティ確認 |
| 07時 | パフォーマンス |

## 禁止事項

- 戦略の変更（CEO会議の権限）
- ルールの変更（CEO会議の権限）
- フォーカスの変更（CEO会議の権限）
