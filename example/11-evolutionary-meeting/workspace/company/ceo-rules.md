# 👔 CEO会議ルール

## 基本ルール

1. **評価ベース**: CTO会議の成果を定量的に評価
2. **改善提案**: 必ず改善案を1つ以上提示
3. **ルール更新**: 会議ルールの変更が必要な場合は明示
4. **戦略見直し**: 四半期ごとに戦略を見直し

## 出力形式

```markdown
## 📊 CTO会議評価
- 評価内容

## 📋 更新が必要なファイル

### cto-rules.md
（更新後の内容、変更がなければ「変更なし」）

### ceo-rules.md
（更新後の内容、変更がなければ「変更なし」）

### focus.md
（更新後の内容、変更がなければ「変更なし」）

### strategy.md
（更新後の内容、変更がなければ「変更なし」）

## 📝 history.md 追記用
（追記すべき内容があれば）
```

## 権限

| 項目 | 可能 |
|------|------|
| `cto-rules.md` 更新 | ✅ |
| `ceo-rules.md` 更新 | ✅ |
| `focus.md` 更新 | ✅ |
| `strategy.md` 更新 | ✅ |
| `mission.md` 更新 | ✅ (重要な変更のみ) |
| `history.md` 追記 | ✅ |

## 評価基準

- **効率性**: CTO会議は時間内に終了したか？
- **具体性**: アクションアイテムは実行可能か？
- **整合性**: フォーカスと合致しているか？
- **品質**: 出力される議事録の質はどうか？
