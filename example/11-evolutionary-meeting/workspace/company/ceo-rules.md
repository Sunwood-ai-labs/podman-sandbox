# 👔 CEO会議ルール

## 基本ルール

1. **評価 → 即時修正**: 評価したら自分でファイルを修正する
2. **継続的改善**: 毎回必ず1つ以上の改善を実施
3. **データドリブン**: CTO会議の成果を定量的に評価

## 実行フロー

```
1. CTO会議評価
   ↓
2. 改善点の特定
   ↓
3. 即座にファイル修正（自分で）
   ↓
4. 結果を議事録に記録
```

## 修正するファイル

CEO会議中に以下のファイルを**直接編集**する：

| ファイル | 更新タイミング |
|----------|----------------|
| `cto-rules.md` | CTO会議のルール改善が必要な時 |
| `ceo-rules.md` | CEO会議自体のルール改善が必要な時 |
| `focus.md` | フォーカスの変更が必要な時 |
| `strategy.md` | 戦略の調整が必要な時 |
| `history.md` | 重要な決定があった時（追記） |

## 出力形式

```markdown
## 📊 CTO会議評価

### スプリントゴール達成率
- 達成率: XX%

### 評価サマリー
- 良かった点
- 改善が必要な点

## 🔧 実施した修正

### [ファイル名] の修正内容
- 変更前: ...
- 変更後: ...
- 理由: ...

## 📝 history.md 追記内容
（追記した内容）

## 📈 次回への引き継ぎ
- 次回のCTO会議に期待すること
```

## 評価基準

| 基準 | 説明 | 重み |
|------|------|------|
| スプリントゴール達成 | 目標を達成したか | 40% |
| インクリメント品質 | 成果物の質 | 30% |
| レトロスペクティブ | 改善意識があるか | 20% |
| ブロッカー解決 | 阻害要因への対処 | 10% |

## 修正の指針

- **小さな改善**: 大きな変更より小さな改善を積み重ねる
- **効果測定**: 変更の効果を次回の評価で確認
- **一貫性**: ミッションと戦略に沿った変更
